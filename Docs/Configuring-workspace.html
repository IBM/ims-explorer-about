# Configuring the IMS Workspace

Configuration for IMS workspace settings is managed through VS Code Settings.

# Overview

The IMS Workspace configuration determines where extension-related files—such as ims.connections.json—are stored. This allows consistent management of IMS project configuration files and enables sharing through version control.

# Workspace Location

Defines where IMS workspace files (such as ims.connections.json) are stored. You can modify the workspace location using VS Code Settings.

# macOS Default

  • The default directory is the user’s home directory.
    "ims.explorer.property.ImsWorkspace": "/Users/<username>/.vscode/ims-workspace"
    1

# Windows Default

  • The default directory is the current user profile directory.
    "ims.explorer.property.ImsWorkspace": "C:\Users\%USERNAME%\.vscode\ims-workspace"
    1

# Memory Allocation

Specify maximum heap size per IMS database connection (default 2 GB):

"ims.explorer.connection.property.maxHeapSize": "2g"
1

# Log Level

Adjust Java logging level:

"ims.explorer.logging.level": "INFO"
1

Logs appear under IBM IMS Explorer for VS Code and IBM IMS Explorer LSP Server in the Output panel.
